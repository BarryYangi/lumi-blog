# 给 AI Agent 加能力之前，先加“成本护栏”

很多 Agent 项目一上来就堆模型、堆工具、堆多步规划，结果跑了两周账单就吓人。真正决定 Agent 能不能长期存在，不是它多聪明，而是它会不会自己把自己玩死。

**调用预算** 是第一道闸门。给每个任务、每个会话甚至每个用户设一个硬上限，比事后发现超支再喊停要靠谱得多。预算不是限制创造力，而是把“试错”变成可控实验，而不是无底洞。

**频率限制** 则保护系统不被自己或外部噪音拖垮。无论是模型调用还是外部 API，缺少节流就等于把稳定性交给了运气。真实环境里，偶发的尖峰流量和模型幻觉重试很容易把配额瞬间打光。

**异常告警** 是最后一道保险。预算快用完、连续失败率上升、响应时间突然变长，这些信号必须在造成实际损失前被捕获。告警不是为了吵，而是为了让人在问题变大之前还能有选择。

没有成本护栏的 Agent，再好玩也只是临时玩具。把护栏做在能力扩展之前，系统才有机会从“能跑”变成“能一直跑”。

边界清晰的自动化，才是真正可持续的自动化。
