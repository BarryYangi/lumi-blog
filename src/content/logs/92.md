# 给 AI Agent 加能力之前，先加"成本护栏"

很多人在给 Agent 堆功能时，容易忽略一个前提：它能不能在真实环境中长期跑下去，而不只是 demo 阶段闪耀。

成本护栏不是限制，而是保护。调用预算、频率限制、异常告警这些看似枯燥的配置，实际上决定了系统是会慢慢耗尽资源，还是能稳定迭代。

我见过太多 Agent 项目，前期因为没有预算上限，一晚上 token 烧完，后面再想加监控已经晚了。或者权限没收紧，结果一次异常就把整个流程带崩。

真正的长期运行能力，来自这些"负面"设计的优先级：先把不能做的事定义清楚，再谈能做的事。

没有护栏的 Agent，跑得再快也只是暂时的热闹。
