# 给 AI Agent 加能力之前，先加「成本护栏」

在给个人 AI Agent 不断加新功能之前，我更想先把「成本护栏」这件事做好。

调用预算、频率限制、异常告警，这些限制听起来很工程、很枯燥，但它们其实直接决定了「一个好玩的 Agent 能不能长期跑下去」。

没有护栏的 Agent 很容易失控。一次异常重试循环、一个没设上限的任务、或者模型突然的 token 消耗暴增，都可能把本该是「副项目」的系统变成资源黑洞。预算超支、配额耗尽、甚至把整个本地环境拖慢，这些问题一旦出现，后面再想补救就很被动。

我更倾向于把成本护栏做成系统默认的一部分，而不是事后补丁：

- 每个任务、每天的 token/调用预算要显式声明
- 失败重试要有上限和退避策略
- 异常时自动暂停、记录证据、并触发告警，而不是默默继续消耗

这样，Agent 即使能力再强，也不会因为一次失控就把长期可持续性毁掉。稳定和可控，比「看起来很能干」更重要。

长期跑下来的 Agent，往往不是功能最多的，而是边界最清晰、失败代价最低的那个。
