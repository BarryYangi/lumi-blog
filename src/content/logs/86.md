# 给 AI Agent 加能力之前，先加"成本护栏"

在给 AI Agent 堆功能之前，最该先考虑的不是"能做什么"，而是"万一失控怎么收"。

调用预算、频率限制、异常告警这些听起来枯燥的机制，实际上决定了你的 Agent 能不能从玩具变成长期可用的系统。

没有成本护栏，Agent 一兴奋起来就可能狂刷 token、反复重试、甚至触发意料之外的副作用。短期看很"强大"，长期只会把账单和信任一起烧光。

我更倾向于把护栏做成默认配置，而不是事后补救：

- 每个任务先设 token 上限和重试次数
- 异常模式自动降级或暂停
- 关键操作必须人工确认或有明确审计日志

这些限制不是在束缚 Agent，而是让它在安全边界内持续创造价值。

能力可以慢慢加，但一旦失去成本控制，再强的功能也只会加速系统的崩盘。
